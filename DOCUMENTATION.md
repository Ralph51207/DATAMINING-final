# Project Documentation: Heart Disease Data Mining

## Overview
This project is a single-notebook data mining pipeline that analyzes heart disease risk using the UCI Heart Disease dataset. It produces descriptive analytics, association rules, patient archetypes, and model evaluation artifacts, then saves outputs into the `outputs/` folder and a report draft.

## Primary Notebook
- `Lastnames_CCS230_Finals.ipynb`

## Data Source
- Dataset: UCI Heart Disease (via `ucimlrepo`)
- Local copy created: `heart.csv`

## End-to-End Workflow
1. **Dependency check**: installs required Python packages if missing.
2. **Data acquisition**: fetches UCI data and normalizes the target to `HeartDisease` (0/1).
3. **Cleaning**: coerces numeric fields, replaces `?` with NaN, and imputes missing values with medians.
4. **EDA**: generates target distribution, histograms, and correlation heatmap.
5. **Association rules**: discretizes numeric features into quartiles, runs Apriori, and filters rules with `HeartDisease=1` as the consequent.
6. **Clustering**: uses K-Means to identify patient archetypes; evaluates K using silhouette score.
7. **Classification**: compares Decision Tree and Random Forest models.
8. **Evaluation**: computes accuracy, precision, recall, and F1; prioritizes recall.
9. **Reporting**: writes CSVs, figures, and a report draft to disk.

## Model Selection Rationale
The evaluation emphasizes **recall** to minimize false negatives in a clinical context. The notebook selects the model with the highest recall (typically Random Forest).
